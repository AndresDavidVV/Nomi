# 🎯 Objetivos y Métricas: Plataforma de Inteligencia Económica

Este documento define el éxito del proyecto desde una perspectiva de negocio y operación. El desarrollo técnico (ver `ROADMAP.md`) debe estar siempre alineado para cumplir estos objetivos.

## 🚀 Objetivos del Proyecto (Versión 1)

1.  **MVP Rápido:** Desplegar un MVP funcional en **~2 semanas** para iniciar captura de información de reuniones.
2.  **Adopción Controlada:** Lograr adopción inicial con **~10 usuarios internos** (funcionarios/directivos y KAMs) como grupo piloto.
3.  **Estandarización:** Eliminar diagnósticos extensos. Usar checklist y campos obligatorios para asegurar completitud mínima ejecutiva.
4.  **Cero Fricción:** Habilitar registro multimodal (texto/audio/fotos) en web app mobile-first con login vía WhatsApp.
5.  **Contexto:** Precargar base de empresas pioneras para identificación rápida.
6.  **Exportabilidad:** Permitir exportación (CSV/Excel) para soportar la evaluación de impacto actual.
7.  **Visión Futura:** Sentar bases para analítica y "conexiones automáticas" (networking inteligente).

## 📊 Objetivos Estratégicos y Métricas (KPIs)

### 1. Activar el Piloto (Ejecución)
*Que el equipo use la plataforma recurrentemente.*
*   **Métrica:** Usuarios activos semanales (WAU) y % de reuniones registradas.
*   **Baseline:** 0 (Desconocido).
*   **Target:** ≥7 WAU (de 10) y ≥60% de sus reuniones registradas.
*   **Plazo:** 30 días post-despliegue.
*   *Por qué:* Sin datos no hay impacto ni futuro matching.

### 2. Asegurar Calidad de Datos (Operación)
*Necesidades capturadas completas y categorizadas.*
*   **Métrica:** % de registros que cumplen checklist completo y % necesidades categorizadas.
*   **Baseline:** 0.
*   **Target:** ≥80% con checklist completo y ≥70% categorizadas.
*   **Plazo:** 60 días post-despliegue.
*   *Por qué:* Habilita la priorización y el matching automático.

### 3. Convertir Necesidades en Acciones (Outcomes)
*Que lo registrado termine en algo concreto.*
*   **Métrica:** % de necesidades con 'acción asignada' y % 'cerrada con evidencia'.
*   **Baseline:** 0.
*   **Target:** ≥50% con acción asignada y ≥25% cerradas.
*   **Plazo:** 90 días post-despliegue.
*   *Por qué:* El impacto real es la intervención ejecutada, no el registro.

### 4. Reducir Tiempo a Solución (Networking)
*Eficiencia en conexiones (ej: embajadas, partners).*
*   **Métrica:** Tiempo promedio (días) desde registro hasta 'conexión realizada'.
*   **Baseline:** ~3 semanas (manual).
*   **Target:** ≤10 días promedio.
*   *Por qué:* Demuestra el valor de la plataforma como acelerador.

### 5. Instalar Medición de Impacto sin Fricción (Madurez)
*Reporting operativo y evaluación interna fluida.*
*   **Métrica:** % de empresas pioneras con exportación generada y frecuencia de revisión mensual.
*   **Baseline:** 0 (Desconocido).
*   **Target:** ≥70% empresas con ≥1 exportación y ≥2 ciclos/mes de revisión por equipo piloto.
*   **Plazo:** 90 días post-despliegue (Q2 2026).
*   *Por qué:* Si la medición es esporádica, se rompe el flujo; la analítica recurrente habilita decisiones.
